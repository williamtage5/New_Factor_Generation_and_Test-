# 数据获取与处理

## 总体目标

- 研究对象：沪深300成分股（`000300.SH`）
- 时间范围：`20150101` 到 `20231231`
- 输出粒度：日频，每个交易日一个 CSV
- 数据来源：Wind
- 核心要求：
  - 时间轴连续（按上交所交易日历）
  - 财务数据按公告日对齐（避免未来函数）
  - 缺失值处理可追溯（打标）

## Step1: 数据获取（SQL）

数据构建思路：
* 先获取交易日期表`ASHARECALENDAR`

* 再映射季度财务数据

1. `请求一段时间的沪深300的PEPB市值.csv`

- 列数：`5`
- `TradeDate` 范围：`20150105` 到 `20231229`
- `TradeDate` 唯一值：`2189`

1. `请求一段时间内的沪深300的季度财务数据ROE、营收增长率、净利润增长率.csv`

## Step2: 数据合并（日频面板）

脚本：`data_processing/build_daily_merged_csv.py`

输入：
- `data/raw_request` 三份 CSV

对齐方法：
1. 以 `OHLC+weight` 为日频主表（主键：`StockCode + TradeDate`）。
2. 估值数据按 `(StockCode, TradeDate)` 左连接。
3. 财务数据按 `AnnounceDate <= TradeDate` 做 as-of 对齐（每只股票取最近公告值）。
4. 输出每个 `TradeDate` 一个 CSV 到 `data/merged_data`。

运行结果：
- 输出目录：`data/merged_data`
- 总行数：`656700`
- 交易日数：`2189`
- 日期范围：`20150105` 到 `20231229`
- 每日行数：最小/最大均为 `300`
- 未来数据泄漏检查（`AnnounceDate > TradeDate`）：`0`

连续性检查：
- `merged_data` 交易日与 `ASHARECALENDAR`（SSE）在该区间完全一致：`2189 / 2189`。一共是8年，日期数据量也差不多。

## Step4：因子生成

1. `momentum_20_raw` = `ClosePrice_t / ClosePrice_{t-20} - 1`  
2. `volatility_20_raw` = `std(ret_1d, 20)`  
3. `roe_raw` = `ROE`  
4. `size_log_mcap_raw` = `log(TotalMarketCap)`  
5. `turnover_proxy_20_raw` = `mean(Amount / TotalMarketCap, 20)`  

6. `momentum_5_raw` = `ClosePrice_t / ClosePrice_{t-5} - 1`  
7. `momentum_60_raw` = `ClosePrice_t / ClosePrice_{t-60} - 1`  
8. `reversal_5_raw` = `-(ClosePrice_t / ClosePrice_{t-5} - 1)`  
9. `downside_vol_20_raw` = `std(min(ret_1d, 0), 20)`  
10. `drawdown_20_raw` = `ClosePrice / rolling_max(ClosePrice, 20) - 1`  

11. `illiq_20_raw` = `mean(abs(ret_1d) / (Amount + 1), 20)`  
12. `ep_ttm_raw` = `1 / PE_TTM`  
13. `bp_inv_raw` = `1 / PB`  
14. `revenue_yoy_raw` = `Revenue_YoY`  
15. `netprofit_yoy_raw` = `NetProfit_YoY`  

前 60 个交易日，部分滚动因子会自然为空。使用第一个非空值进行填充。

处理结果是：
* 日期范围：20150105 到 20231229
* 天数：2189
* 总行数：656700
* 每日 300 只：通过（daily_rows_bad_count=0）
* 因子列缺失（填充后）：0
* 全表总缺失：0


## Step5：处理异常值

**1. 输入与输出**
1. 输入：`data/factor_generation`
2. 输出：`data/handled_outliers`（每天一个 CSV）
3. 覆盖策略：重跑时先清空 `data/handled_outliers` 下旧 CSV 再写新文件

**2. 处理对象**
仅对 15 个因子列做异常值处理（原始基础字段不改）：
- `momentum_5_raw`
- `momentum_20_raw`
- `momentum_60_raw`
- `reversal_5_raw`
- `volatility_20_raw`
- `downside_vol_20_raw`
- `drawdown_20_raw`
- `turnover_proxy_20_raw`
- `illiq_20_raw`
- `roe_raw`
- `size_log_mcap_raw`
- `ep_ttm_raw`
- `bp_inv_raw`
- `revenue_yoy_raw`
- `netprofit_yoy_raw`

**3. 3σ处理规则（按日横截面）**
对每个 `TradeDate`、每个因子：
1. 计算当日均值 `mu`、标准差 `sigma`
2. 上下界：`[mu - 3*sigma, mu + 3*sigma]`
3. 超出边界的值截断到边界（winsorize）
4. 若 `sigma=0` 或样本不足（例如 <10），该日该因子跳过不处理
